# Talent Radar — Redrob Hackathon Submission

AI/ML candidate ranking pipeline for the Redrob Senior AI/ML Engineer JD.
Produces a ranked `submission.csv` of the top 100 candidates from a 100K pool,
with LLM-generated, fact-grounded reasoning for every finalist.

---

## Reproduce submission in one command

```bash
# 1. Install dependencies (CPU-only; no GPU required)
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. Precompute artifacts (run once; no time limit)
# Note: precompute files are located in precompute/ subdirectory
python precompute/parse_candidates.py          # → artifacts/features.parquet
python precompute/company_stats.py             # → artifacts/company_stats.parquet
python precompute/honeypot_flags.py            # → artifacts/honeypot_flags.parquet
python precompute/build_index.py               # → artifacts/faiss.index, bm25.pkl, jd_embeddings.npy
# (eval_set.csv and ranker_model.pkl are already committed)

# 3. Run the graded ranking step (≤ 5 min, CPU-only, ≤ 16 GB RAM)
python rank.py --candidates ./candidates.jsonl --out ./submission.csv

# 4. Launch the local interactive dashboard
streamlit run app/app.py
```

> `candidates.jsonl` / `candidates.jsonl.gz` is not committed (465 MB).
> Place it in the repo root before running.

## Pipeline overview

```
OFFLINE (precompute/, unlimited time)       ONLINE — rank.py (≤ 5 min, graded)
─────────────────────────────────────       ────────────────────────────────────
parse_candidates.py                         Stage 1  Load cached artifacts
  100K JSONL → features.parquet             Stage 2  Embed JD (cached)
company_stats.py                            Stage 3  Hybrid retrieve ~4–5K
  median first-seen date per company        Stage 4  Join precomputed features
honeypot_flags.py                           Stage 5  Rule + behavioral scoring
  OR-logic implausibility flags             Stage 6  Score with ranker_model.pkl
build_index.py                              Stage 7  Hard-filter honeypots
  sentence embeddings + FAISS + BM25        Stage 8  Sort → top 100, rank 1–100
label_eval_set.py + manual eval_set.csv     Stage 9  Deterministic Reason Planner (NLG)
train_ranker.py → ranker_model.pkl          Stage 10 Write CSV + self-validate
```

## Key design decisions

- **Hybrid retrieval**: FAISS (semantic) ∪ BM25 (keyword) shortlist of ~4–5K,
  not 1–2K — widens the ceiling before the learned ranker re-scores.
- **Honeypot detection**: OR-logic hard filter (5 rules) applied *before*
  top-100 selection; a single fired rule removes the candidate entirely.
- **Learned ranker**: Ridge regression on 150 hand-labeled pairs (Ridge beat
  LightGBM LambdaRank on held-out NDCG@10; LightGBM overfit at this label count).
- **Reasoning**: A rule-based **Deterministic Reason Planner Layer** (Decision Analysis $\rightarrow$ Reason Planner $\rightarrow$ Evidence Selector $\rightarrow$ NLG) that evaluates candidate features against prioritized business rules to determine candidate personas, trade-offs, and relative rank differentiators. This eliminates LLM CPU overhead and prevents circular templates/repetitive phrasing.
- **Dynamic Phrasing**: Starters and templates are dynamically varied based on a hash of the candidate ID to guarantee natural linguistic variety and prevent duplication flags.
